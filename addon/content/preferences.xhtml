Obsidian vault path Browse… Absolute path to your vault. Used for the “Open in Obsidian” link. Notes folder Browse… Folder (inside the vault) where literature notes live and are created. Templates folder Browse… Install starter templates… Folder of templates. note.md (and any note-*) are whole-note scaffolds; every other file is an insertable block template (shown in the Template dropdown by filename). See TEMPLATES.md in this folder for how to write one. Install starter templates… copies the plugin's defaults into this folder (existing files are kept). Even with no folder set, the built-in templates work as a fallback. Default note template Used by “Create note from template”. Can be any template — a whole-note scaffold or a per-annotation/field template (then a created note is just that block). Filename pattern New-note filename, so it can match how your vault already names notes. Any of these placeholders work: {{citekey}}, {{author}} (first author's surname), {{year}}, {{title}}, {{journal}}, {{date}}, {{itemType}}. Examples: @{{citekey}}.md · {{author}} {{year}} - {{title}}.md · {{year}} {{title}}.md. The .md is added if you omit it, and characters illegal in filenames are removed automatically. Tag sync field Default frontmatter field mirrored to Zotero tags by ⋯ More → Push tags → Zotero (e.g. Topics). A note can override this — once you push, the chosen field is recorded in the note's zon: map, so it's per-note. [beta] reverse sync; you confirm every change before it's written to Zotero. Image-annotation folder Vault-relative folder where image (area) annotations are exported on sync. Each item gets its own subfolder by citekey (e.g. References/Attachments/{{citekey}}/), and the note links them with ![[…]] embeds — which render in Obsidian. This is the default; a note can override it with zon: attachments: in its frontmatter. (Ink/drawing annotations aren't exported yet.) Auto-sync annotations while reading When on, an open note's live annotation blocks refresh automatically as you highlight in the PDF — no need to click Refresh. Only notes you have open are touched, and your prose is never changed. Also toggleable from the note pane toolbar. Enable experimental features Adds a ⋯ More menu to the note pane with early/advanced actions: Sync Metadata, Migrate, and Push tags → Zotero (reverse sync). Off by default to keep the toolbar tidy. Reopen the item (or restart Zotero) for the change to take effect. Legacy fallbacks (used only if the Templates folder is unset or missing the relevant file): Note template file (legacy) Fallback whole-note template, used if the Templates folder has no note.md. Custom formats folder (legacy) Optional older folder of block templates; merged in, but the Templates folder wins on name clashes.
